DOCX Research Reports
====================
Author: Dr Yuqian Zhang
Date: 16 July 2026

This folder contains academic journal-style Word (.docx) versions of HTML research
reports, structured for submission to SSRN and similar open-access repositories.

Reports are auto-discovered from two locations:
  - public/reports/<topic-slug>/index.html   (canonical; preferred)
  - <project-root>/*_research_brief.html      (legacy; fallback)
  - <project-root>/<topic-subfolder>/<topic>.html  (legacy; fallback)

When a report exists in both public/reports/ and the project root, only the
public/reports/ version is used.

Automation
----------
A scheduled task runs daily at 06:00 NZST. The sync_reports.py script detects all
reports automatically by scanning the project structure. Only reports whose source
HTML has changed are regenerated.

To add a new report:
  Place index.html inside public/reports/<your-topic-slug>/.
  The next sync will detect and convert it automatically.

Manual Sync
-----------
  python3 reports-docx/sync_reports.py

Pipeline
--------
  1. sync_reports.py     - Auto-discovers reports, writes manifest, orchestrates
  2. extract_html.py     - Parses HTML, extracts structured text into JSON
  3. generate_docx.js    - Creates .docx with academic formatting from JSON
  4. strip_watermark.py  - Removes any "Generated by AI" watermark metadata
